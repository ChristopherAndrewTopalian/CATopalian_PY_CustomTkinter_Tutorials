# Installing CustomTkinter & The Dark-Theme Philosophy

Welcome to Day 1 of learning interface building with CustomTkinter! 

In this series, we are taking a **procedural, top-to-bottom approach**. If you have ever written HTML to build a web page, you will feel right at home. We will not be using complex Object-Oriented Programming (OOP) or confusing `class` keywords, just simple, readable code that executes step-by-step from top to bottom.

## Why CustomTkinter over Standard Tkinter?

Standard Python includes a GUI library called Tkinter. While powerful, it defaults to a very bright, 1990s-era interface that blasts your eyes with glaring white backgrounds (`#FFFFFF`). Staring at high-intensity white screens for hours causes severe visual fatigue and eye strain.

**CustomTkinter** is a modern wrapper around Tkinter that gives us two massive advantages:
1. **Hardware-Friendly Dark Themes by Default:** We can turn the entire application into a soothing, low-contrast charcoal theme with a single line of code.
2. **Modern Visuals:** Rounded corners, smooth borders, and clean fonts work automatically without writing dozens of lines of styling code.

---

## Step 1: Open Your Terminal

Before we can write any Python code, we need to download the CustomTkinter library to our computer.

1. On **Windows**, press `Win + R`, type `cmd`, and press **Enter** to open the Command Prompt.
2. On **Mac/Linux**, open your standard **Terminal** application.

---

## Step 2: Install the Library

Type the following command into your terminal and press **Enter**:

```bash
pip install customtkinter
```
